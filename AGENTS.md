> Audience: any LLM-based coding agent reading this repo as context

## What this is

Multi-host, portable NixOS flake. Single primary user (`oonishi`).
Decomposed in three layers so the same configuration runs across
laptops, desktops, servers, VMs without copy-paste:

```
modules/nixos/        universal NixOS layer — works on any x86_64 host
modules/hardware/     opt-in profiles per CPU / GPU / form-factor / boot
hosts/<name>/         host composition — picks profiles + adds quirks
```

A host is just an attribute under `nixosConfigurations` that picks a
list of modules. Adding a new device means creating
`hosts/<name>/default.nix`, dropping a `hardware-configuration.nix`
generated by `nixos-generate-config` next to it, and importing the
appropriate `modules/hardware/*` profiles.

## Conventions

### Commits
Conventional Commits, lowercase, imperative mood, max 72 chars on the
first line:

    <type>(<scope>): <short description>

Types: `feat` (new module/capability), `fix` (bug or rebuild error),
`refactor` (no behaviour change), `style` (formatting), `docs`, `chore`
(version bumps, GC, lock updates).

Scope: module name without `.nix`, or `flake`, `secrets`, `hosts`.

### Code style
- Formatter: `alejandra` (auto-applied by pre-commit).
- Linter: `statix` with `statix.toml`.
- Dead code: `deadnix --no-lambda-pattern-names --fail`.
- Diagnostics: `nil diagnostics <file>` (eval-time errors).
- Secrets: `gitleaks git --pre-commit --staged` (no plaintext leaks).
- Module function signature lists ONLY the arguments the body actually
  uses. Use `_:` for an empty pattern.
- Booleans: `enable = true;` / `enable = false;` — never bare.
- Use `lib.mkIf` for conditional blocks, not Nix `if … then … else`.
- `lib.mkForce` only when overriding upstream defaults; `lib.mkDefault`
  for values downstream might override.
- `with pkgs;` only inside list literals, never in scope of expressions.

### Required pre-commit
Five hooks — every commit must pass all five. They run automatically
via `.pre-commit-config.yaml`.

```
alejandra .
statix check --config statix.toml
deadnix --no-lambda-pattern-names --fail
nil diagnostics <files>
gitleaks git --pre-commit --staged
```

`--no-verify` is forbidden.

## Workflows

### Verify a change
```
sudo nixos-rebuild test --flake /etc/nixos#lecoo --show-trace 2>&1 | tail -80
```

### Apply a change (creates a boot menu entry)
```
sudo nixos-rebuild switch --flake /etc/nixos#lecoo --show-trace 2>&1 | tail -80
```

### Add a new application
1. Verify the package/option name before editing. Letta Code agents use
   `nixos-options` first (`nixos_info` / `nixos_search`); other agents
   fall back to `nix search nixpkgs <name>` if they have no MCP/skill
   access. Do not guess attribute or option paths.
2. If present:
   - System-wide: add to `modules/nixos/packages.nix`.
   - Per-user with HM module: create or extend `home/<name>.nix`.
   - Per-user without HM module: create a small `home/<name>.nix` that
     adds the package to `home.packages`, then import it from
     `home/default.nix`.
3. If absent in nixpkgs but available via NUR: use `pkgs.nur.repos.<...>`.
4. If only as binary/AppImage/prebuilt archive: create
   `pkgs/<name>/default.nix`, register it in `pkgs/default.nix`, and
   document non-obvious packaging traps in comments.
5. Run the verification sequence (alejandra → statix → deadnix → nil → gitleaks → rebuild test).

NEVER `nix-env -i`, `pip install --user`, `npm install -g`,
`apt`/`pacman`/anything imperative. Everything declarative.

### Add a new secret
1. List authorised public keys in `secrets/secrets.nix`.
2. `cd secrets && agenix -e <name>.age` — opens `$EDITOR`, save plain
   text, file gets re-encrypted.
3. Declare consumption in `modules/nixos/secrets.nix`:
   ```nix
   age.secrets.<name> = {
     file = ../../secrets/<name>.age;
     owner = username;
     mode = "400";
   };
   ```
4. Reference `config.age.secrets.<name>.path` from the consumer module
   (default location: `/run/agenix/<name>`).

Decryption identity is the host's `/etc/ssh/ssh_host_ed25519_key`. Never
commit any plaintext key, and never hand-edit a `.age` file.

### Add a new host
1. `mkdir hosts/<name> && cd hosts/<name>`.
2. Boot the target machine off a NixOS installer USB.
3. `nixos-generate-config --root /mnt --dir .` — copy
   `hardware-configuration.nix` into `hosts/<name>/`.
4. Create `hosts/<name>/default.nix` listing the right
   `modules/hardware/*` profiles and `system.stateVersion`.
5. Register in `flake.nix`:
   ```nix
   nixosConfigurations.<name> = mkHost {
     hostname = "<name>";
     username = "<user>";
     system   = "x86_64-linux";
     modules  = [ ./hosts/<name> ];
   };
   ```
6. Append the new host's SSH host pubkey to `secrets/secrets.nix`,
   re-encrypt all required `.age` files (`agenix -r`).
7. `nixos-install --flake .#<name>`.

## Documentation maintenance

When a change affects workflows, commands, invariants, repository layout,
agent contract, tool/runtime wiring, or anything described in `README.md`
or this file, update `README.md` and/or `AGENTS.md` in the same change set
before pushing.

Also update narrower docs when applicable:
- User-visible behaviour → `docs/handbook.md`.
- Architecture or module graph → `docs/architecture.md`.
- Code style or Git rules → `docs/conventions.md`.
- Step-by-step procedures → `docs/workflows.md`.
- Module-level reasoning → the header comment of the relevant `.nix` file.

## Boundaries

These are hard rules. Violations are bugs, not feature requests.

- Do not modify `hosts/lecoo/hardware-configuration.nix` unless re-running
  `nixos-generate-config` on the actual hardware.
- Do not edit anything under `/home/<user>/.config/<app>/` directly —
  these paths are HM-managed (symlinks into `/nix/store`). Edit the
  relevant `home/<app>.nix` module instead.
- Do not commit secrets in plaintext. The only secrets in the repo are
  `secrets/*.age` (encrypted).
- Do not push to `main` without `nixos-rebuild test` passing.
- Never `git rebase -i`, `git reset --hard`, force-push, or skip hooks
  (`--no-verify`) without explicit user approval in this conversation.
- Never change `system.stateVersion`.
- Never `allowUnfree` per-package — it's set globally in `modules/nixos/nix.nix`.
- Never set `services.openssh.settings.PasswordAuthentication = false`
  without explicit user approval.
- Lecoo ultra-economy contract: any change touching Hyprland monitor
  names/modes, refresh-rate policy, Waybar host widgets,
  `power-profiles-daemon`, `lecoo-ctrl`, AMDGPU DPM/sysfs paths, AC
  adapter sysfs naming, or the laptop hardware profile must review
  `hosts/lecoo/home/scripts.nix` (`ultra-economy-toggle`) and
  `hosts/lecoo/ec.nix` (`lecoo-sync-power-profile`). The toggle assumes
  eDP-1 2880x1800 120↔60 Hz, `ADP1`, and
  `/sys/class/drm/card1/device/power_dpm_force_performance_level`; if
  those contracts change, update the toggle in the same patch.
